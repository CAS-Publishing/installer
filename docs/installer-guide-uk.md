## 1. Що це і навіщо

Інсталер — це Editor-інструмент, який зводить інтеграцію реклами, атрибуції та аналітики до
кількох кліків. Він:

- сам прописує потрібні scoped-реєстри в `manifest.json`;
- встановлює компоненти у перевірених сумісних версіях **через UPM** (єдиний метод встановлення);
- знаходить старі установки (legacy npm-id, імпортовані `.unitypackage`, git-URL) і мігрує їх на
  UPM, не створюючи дублікатів;
- показує готовність налаштування по кожній платформі та веде вас у потрібне місце (нативні
  налаштування CAS.AI, консоль Firebase) — сам файлів конфігурації більше не пише;
- оновлює сам себе та свій каталог пакетів у фоні.

Список пакетів і правила міграції живуть в окремому data-пакеті
`com.psvgamestudio.installer.metadata` — нові компоненти й правила доїжджають до проєкту без
оновлення самого інсталера.

**Основні компоненти:**

| Компонент | Package id | Призначення |
|---|---|---|
| **CAS SDK** | `com.cleversolutions.ads.unity` | Реклама / медіація |
| **Tenjin SDK** | `com.tenjin.sdk` | Атрибуція |
| **Firebase Analytics** | `com.google.firebase.analytics` (+ `firebase.app` і EDM транзитивно) | Аналітика |
| **EDM4U** | `com.google.external-dependency-manager` | Резолв нативних Android/iOS-залежностей |

Крім основних, на вкладці **Components** є розділ **Additional components** — адаптери на базі
`com.psv.core` (`PSV Analytics (Firebase)`, `PSV Remote Config (Firebase)`) та `PSV Tenjin
(Attribution)`. Адаптери Firebase з'являються в списку, лише коли в проєкті вже є `com.psv.core`.

## 2. Встановлення інсталера

Є три способи отримати сам інсталер. Який би ви не обрали — далі все однаково: інсталер сам
підтягне каталог метаданих і відкриє **CAS.AI Publishing Hub**.

> Вручну ви реєструєте лише scope `com.psvgamestudio` (спосіб A). Реєстри для CAS / Tenjin /
> Firebase (`com.cleversolutions`, `com.tenjin`, `com.google`) інсталер додає **сам**, коли
> встановлює відповідний компонент.

### Спосіб A — UPM scoped registry (основний)

**A1 — через Unity UI:**

1. Додайте реєстр: `Edit → Project Settings → Package Manager → Scoped Registries → +`

   | Поле | Значення |
   |---|---|
   | Name | `PSV Game Studio` |
   | URL | `https://npm.psvgamestudio.com/` |
   | Scope(s) | `com.psvgamestudio` |

   Натисніть **Save / Apply**.

2. Додайте пакет: `Window → Package Manager → + → Add package by name…`

   ```
   com.psvgamestudio.installer
   ```

   Поле версії лишіть порожнім (буде остання) або вкажіть конкретну, наприклад `0.0.1-preview.39`.

**A2 — редагуванням `manifest.json`** (той самий результат):

```jsonc
{
  "scopedRegistries": [
    {
      "name": "PSV Game Studio",
      "url": "https://npm.psvgamestudio.com/",
      "scopes": [ "com.psvgamestudio" ]
    }
  ],
  "dependencies": {
    "com.psvgamestudio.installer": "0.0.1-preview.39"
  }
}
```

Збережіть файл — Unity сам зарезолвить інсталер, коли вікно редактора отримає фокус.

### Спосіб B — Git URL (швидка проба / CI)

Реєстр не потрібен — пакет дзеркалиться на публічний GitHub:

`Window → Package Manager → + → Add package from git URL…`

```
https://github.com/CAS-Publishing/installer.git#0.0.1-preview.39
```

Без суфікса `#<версія>` Unity братиме останній реліз із гілки за замовчуванням.

> ⚠️ Git-установка інсталера **не самооновлюється** з вкладки About — щоб перейти на новішу
> версію, змініть `#<версія>` в URL. Компоненти при цьому все одно встановлюються через UPM.

### Спосіб C — bootstrap `.unitypackage` (нуль налаштувань)

Для проєктів, де не хочеться пояснювати команді про реєстри, є крихітний
`PSVInstaller-Bootstrap.unitypackage`:

1. `Assets → Import Package → Custom Package…` → оберіть `PSVInstaller-Bootstrap.unitypackage`.
2. При імпорті він сам пропише реєстр і залежність у `manifest.json` (попередньо зробивши копію
   `manifest.json.bak`) і запустить resolve. Інсталер підхопиться, поставить метадані й відкриє
   Hub.
3. Коли все на місці, bootstrap **самовидаляється** — у проєкті від нього нічого не лишається.

## 3. Перший запуск — CAS.AI Publishing Hub

Після встановлення інсталер за один-два перезавантаження доменів (це нормально — Unity
перекомпільовується після кожного нового пакета) відкриє вікно **CAS.AI Publishing Hub**
автоматично. Якщо ні — меню:

```
CAS Hub → Open Hub
```

(або `Assets → CleverAdsSolutions → Hub`).

На абсолютно новому проєкті вікно веде вас через короткий інтро-флоу з чотирьох кроків; після
нього ви більше сюди не повертаєтесь — вікно відкривається одразу на вкладках (розділ 4).

### Крок 1 — Ready

Список основних компонентів (CAS SDK, Tenjin SDK, Firebase Analytics, EDM4U) з поточним станом
кожного та кнопка **Install**, яка ставить усе, чого ще немає. Якщо все вже встановлено, кнопка
одразу веде на крок **Configuration**. Є посилання **Advanced**, яке пропускає авто-встановлення
і одразу відкриває вкладку **Components** для вибіркового налаштування.

### Крок 2 — Progress

Компоненти встановлюються **по одному**; між кроками Unity перезавантажує домени — прогрес це
переживає, нічого натискати не треба.

### Крок 3 — Configuration

Той самий екран готовності, що описаний у розділі 4 (вкладка **Configuration**) — тут він показаний
як крок майстра. **Continue** розблоковується, щойно хоча б **одна** платформа (Android або iOS)
повністю налаштована — багато проєктів виходять лише на одній платформі, і цього достатньо.

### Крок 4 — Done

Короткий чек-лист (CAS / Tenjin / Firebase) для активної платформи білду — суто інформаційний, нічого
не блокує. Кнопки ведуть на вкладку **Components** або закривають вікно.

> Якщо на старті з'явився діалог «каталог ще встановлюється» — просто зачекайте: після
> перезавантаження Unity встановлення продовжиться саме, повторний клік не потрібен.

Повторно пройти інтро (наприклад, щоб показати флоу колезі або переналаштувати з нуля) можна через
`Assets → CleverAdsSolutions → Hub (Restart Intro)`.

## 4. Повсякденна робота: вкладки

Після завершення інтро вікно працює як трьохвкладковий інструмент: **Components** /
**Configuration** / **About**.

### Components

**Main components** — той самий список, що й на кроці Ready, з розширеними діями. Нижче —
**Additional components**: адаптери каталогу (`PSV Analytics (Firebase)`, `PSV Remote Config
(Firebase)` — з'являються тільки коли в проєкті вже є `com.psv.core`; `PSV Tenjin (Attribution)`).

| Стан | Що означає | Дія в рядку |
|---|---|---|
| 🟢 Up to date | стоїть актуальна версія | — |
| 🟡 Update required | у каталозі є новіша версія | **Update** |
| 🟡 Manual install | знайдена стара установка (legacy / `.unitypackage` / git) | **Connect to Hub** (для промотованої legacy-групи кнопка називається **Migrate**) |
| 🟡 Mixed install | суперечливий стан (одразу кілька джерел) | **Fix** |
| 🟡 Needs registry | пакет стоїть, але не вистачає scoped-реєстру | **Fix** |
| 🔴 Not installed | компонента немає | **Install** |

Кнопка **Refresh** внизу перечитує стан проєкту й каталог з диска; статуси також оновлюються
автоматично одразу після будь-якої дії в рядку.

> Якщо SDK було колись імпортовано `.unitypackage`-ом, кнопки **Install** ви не побачите — буде
> **Connect to Hub**. Це захист від дублікатів (дві копії SDK ламають проєкт).

**Компаунд-міграція Firebase.** Якщо в проєкті ще стоїть стара обгортка `com.psv.firebase.base`,
і рядок Firebase, і рядки адаптерів (`PSV Analytics` / `PSV Remote Config`) пропонують **Migrate**,
яка одним кроком знімає legacy-пакет і ставить нативний Firebase разом із потрібними адаптерами.

### Configuration

Read-only таблиця готовності **по обох платформах (Android / iOS)** для всіх встановлених
компонентів. Клітинки клікабельні:

- **CAS** та **Tenjin** — відкривають нативне вікно налаштувань **CAS.AI** для вибраної платформи
  (саме там тепер живуть CAS App ID, ad formats, мережі медіації та Tenjin key — інсталер сам
  більше нічого в ці налаштування не пише).
- **Firebase** — якщо файл конфігурації вже знайдено, клітинка підсвічує його в Project-вікні;
  якщо ні — відкриває діалог **Locate file…**.

Під таблицею з'являються картки з підказками для того, що ще не готове (CAS, Tenjin key, Firebase
config file), з прямими кнопками-переходами.

Якщо на Android-проєкті бракує gradle/manifest-шаблонів, угорі з'явиться банер із кнопкою
**Enable Android build settings** — натисніть, інакше білд може не збиратися.

**Continue** розблоковується, щойно хоча б одна платформа повністю налаштована.

### About

Поточна версія інсталера + перевірка реєстру на новішу. Червона крапка на вкладці = доступне
оновлення. Кнопка **Update** оновлює інсталер на місці (тільки для UPM-установки; git-установка
оновлюється зміною `#тега` в URL).

## 5. Налаштування CAS

Інсталер **більше не пише** CAS-налаштування сам — усе робиться у нативному вікні налаштувань
**CAS.AI**, яке відкривається кліком по клітинці CAS/Tenjin на вкладці **Configuration** (або
кнопкою у картці-підказці):

- **CAS App ID** — окремо для Android та iOS, у самому вікні CAS.AI.
- **Ad formats** (Banner / Interstitial / Rewarded / App Open) та вибір **мереж медіації**
  (Optimal / Children Ads) — там само.
- **Tenjin SDK Key** — теж у вікні CAS.AI; вкладка **Configuration** лише показує, чи він заданий,
  і веде до потрібного вікна, якщо ні.

Вкладка **Configuration** — це індикатор готовності, а не редактор: вона показує зелений/жовтий
стан і, за потреби, відкриває правильне нативне вікно, але значення туди не пише.

## 6. Налаштування Firebase

Інсталер ставить пакети Firebase, але **конфіг-файли ви додаєте самі** (вони унікальні для
вашого застосунку):

| Платформа | Файл | Звідки взяти |
|---|---|---|
| Android | `google-services.json` | Firebase Console → Project settings → ваш Android-застосунок |
| iOS | `GoogleService-Info.plist` | Firebase Console → Project settings → ваш iOS-застосунок |

Покладіть файл у `Assets/` — вкладка **Configuration** показує, знайдений він чи ні; клік по
клітинці з відсутнім файлом відкриває діалог **Locate file…**, а якщо файл уже на місці — клік
підсвічує його в Project-вікні.

EDM (External Dependency Manager) — тепер окремий компонент у списку **Main components**
(`com.google.external-dependency-manager`); ставиться й оновлюється як і решта.

## 7. Міграція старих установок

Якщо в проєкті вже був CAS / Tenjin / Firebase, встановлений «по-старому», інсталер це помітить і
запропонує **Connect to Hub** (для промотованої legacy-групи — **Migrate**):

- **Legacy npm-id / legacy-файли в `Assets/`** → видаляється старе, ставиться канонічний
  UPM-пакет, реєструється scope.
- **`.unitypackage`-установка** → строго у два кроки: спершу видаляється ручна копія, і лише після
  успіху ставиться UPM-пакет (щоб не отримати дублікат).
- **Git-установка компонента** → та сама кнопка на вкладці **Components** переписує git-залежність
  на реєстрову.
- **Legacy `com.psv.firebase.base`** → компаунд-міграція: один клік знімає стару обгортку й ставить
  нативний Firebase разом із адаптерами (`PSV Analytics` / `PSV Remote Config`), пропонується як з
  рядка Firebase, так і з рядків адаптерів.

Перед видаленням завжди показується вікно підтвердження з **точним списком папок**, які будуть
видалені. Правила безпеки:

- Видаляються **лише** папки SDK, оголошені в каталозі (`CleverAdsSolutions`, `Tenjin`,
  `Firebase`, `ExternalDependencyManager`, `PlayServicesResolver`…). Ваші папки
  (`Assets/Scripts` тощо) під видалення потрапити не можуть.
- Спільні папки (`Assets/Plugins` і подібні) автоматично **не** видаляються — про них буде окреме
  попередження «видаліть вручну».
- **Git-guard:** якщо у файлах, що видаляються, є незакомічені зміни — міграція відмовить із
  проханням спершу закомітити. Так завжди є шлях відкату: `git restore .`.

> **Порада (і джунам, і сіньйорам): комітьте проєкт перед міграцією.** Інсталер сам про це
> подбає git-гардом, але чиста робоча копія робить будь-який відкат тривіальним.

## 8. Оновлення

| Що | Як оновлюється |
|---|---|
| **Компоненти** (CAS, Tenjin, Firebase, EDM4U, адаптери) | Вкладка **Components** → кнопка **Update** у рядку, коли в каталозі з'являється новіша рекомендована версія. |
| **Інсталер** | Вкладка **About** → **Update** (сам себе через UPM). Git-установка — зміною `#тега`. |
| **Каталог метаданих** | Автоматично й тихо у фоні при завантаженні Editor. Нові пакети та правила міграції з'являються без ваших дій. |

Версії компонентів завжди беруться з каталогу (перевірені сумісні пари) — не оновлюйте CAS /
Firebase / Tenjin вручну через Package Manager в обхід інсталера.

## 9. Розв'язання проблем

**Hub не відкрився після встановлення.**
Дочекайтеся, поки Unity завершить перезавантаження доменів (на першому імпорті їх може бути
кілька). Якщо вікна все одно немає — відкрийте вручну: `CAS Hub → Open Hub`. У консолі шукайте
рядки з префіксом `[PSV Installer]` / `[CAS Hub]` — вони пояснюють, на якому кроці процес.

**«Recommended install» каже, що каталог ще встановлюється.**
Це нормально на найпершому запуску: каталог — окремий пакет, який ще резолвиться. Натисніть OK і
зачекайте — після перезавантаження Unity встановлення продовжиться автоматично.

**Кнопка Install «крутиться» довго.**
Кожен компонент — це UPM-resolve + перезавантаження доменів; на великому проєкті крок може
тривати хвилину-дві. Вбудований watchdog не дасть процесу зависнути назавжди — у разі реального
стопу в консолі з'явиться пояснення з префіксом `[PSV Installer]`.

**Міграція відмовляє: «uncommitted changes / закомітьте спершу».**
Так і задумано (git-guard). Закомітьте або приберіть зміни у файлах SDK і повторіть. Відкат після
міграції — `git restore .`.

**Configuration показує жовті пункти.**
Клікніть по проблемній клітинці — вона відкриє нативне вікно налаштувань CAS.AI або діалог
пошуку Firebase-файлу. Найчастіші причини: не покладений `google-services.json` /
`GoogleService-Info.plist`, не введений CAS App ID / Tenjin key для цієї платформи в CAS.AI, не
ввімкнені Android build templates (банер угорі з кнопкою-фіксом).

**Unity 6.3 скаржиться на підпис пакета (invalid/missing signature).**
Пакети з нашого реєстру публікуються підписаними. Unity 6000.3 версій **до 6000.3.5f2** має ваду,
через яку валідні підписи позначаються як invalid — оновіть патч Unity перед тим, як бити на
сполох.

**Треба почати налаштування заново.**
`Assets → CleverAdsSolutions → Hub (Restart Intro)` — прожене Ready → Progress → Configuration →
Done заново. На вже встановлені пакети це не впливає.

## 10. Гарантії безпеки

- **Editor-only** — жоден код інсталера не потрапляє у білд гри.
- **`manifest.json` руками не редагується** — всі правки інсталер вносить сам та ідемпотентно.
- **CAS-налаштування інсталер сам не пише** — CAS App ID, ad formats, мережі та Tenjin key
  задаються виключно в нативному вікні CAS.AI.
